---
title: Documents
description: Configuring which files contain GraphQL operations and fragments.
---

The `documents` field tells GraphQL Analyzer where to find your GraphQL operations and fragments.

## Basic patterns

```yaml
# Pure GraphQL files
documents: "src/**/*.graphql"

# Multiple file types
documents: "src/**/*.{graphql,ts,tsx,vue,svelte,astro}"
```

## Multiple patterns

```yaml
documents:
  - "src/**/*.graphql"
  - "src/**/*.gql"
  - "src/**/*.tsx"
  - "src/**/*.ts"
  - "src/**/*.jsx"
  - "src/**/*.js"
  - "src/**/*.vue"
  - "src/**/*.svelte"
  - "src/**/*.astro"
```

## Excluding files

Use `!` prefix to exclude patterns:

```yaml
documents:
  - "src/**/*.{ts,tsx}"
  - "!src/**/*.test.ts"
  - "!src/**/*.spec.ts"
```

## Embedded GraphQL

When you include `.ts`, `.tsx`, `.js`, `.jsx`, `.vue`, `.svelte`, or `.astro` files, GraphQL Analyzer automatically extracts GraphQL from tagged template literals:

```typescript
import { gql } from "graphql-tag";

const query = gql`
  query GetUser($id: ID!) {
    user(id: $id) {
      id
      name
    }
  }
`;
```

For Vue, Svelte, and Astro files, the analyzer first extracts `<script>` blocks (or Astro frontmatter), then runs the same extraction pipeline. See [Embedded GraphQL](/graphql-analyzer/ide-features/embedded-graphql/) for details and examples.

### Configuring extraction

By default, the tool recognizes `gql` imported from common modules. You can customize this:

```yaml
documents: "src/**/*.tsx"
extensions:
  graphql-analyzer:
    extractConfig:
      tagIdentifiers: ["gql", "graphql", "query"]
      modules: ["graphql-tag", "@apollo/client", "custom-gql"]
      allowGlobalIdentifiers: true
```

| Option                   | Default                                                                                                | Description                                                                                                  |
| ------------------------ | ------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------ |
| `tagIdentifiers`         | `["gql", "graphql"]`                                                                                   | Tag function names to recognize                                                                              |
| `modules`                | `["graphql-tag", "@apollo/client", "apollo-server", "apollo-server-express", "gatsby", "react-relay"]` | Import sources to recognize                                                                                  |
| `magicComment`           | `"GraphQL"`                                                                                            | Enables extraction from string literals and untagged template literals preceded by a `/* GraphQL */` comment |
| `allowGlobalIdentifiers` | `false`                                                                                                | Allow unimported tag identifiers                                                                             |
