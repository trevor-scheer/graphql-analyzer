---
title: Linting Configuration
description: How to configure lint rules, presets, and severity levels.
---

## Using a preset

The simplest configuration:

```yaml
extensions:
  graphql-analyzer:
    lint: recommended
```

## Preset with overrides

```yaml
extensions:
  graphql-analyzer:
    lint:
      extends: recommended
      rules:
        noDeprecated: off # Disable a recommended rule
        requireIdField: error # Enable an additional rule
```

## Rules only (no preset)

```yaml
extensions:
  graphql-analyzer:
    lint:
      rules:
        uniqueNames: error
        noDeprecated: warn
```

## Severity levels

| Level   | Description      | Affects exit code? |
| ------- | ---------------- | ------------------ |
| `error` | Shown as error   | Yes (exit 1)       |
| `warn`  | Shown as warning | No                 |
| `off`   | Rule disabled    | —                  |

## Rule options

Some rules accept additional configuration via array or object syntax:

```yaml
# Array syntax: [severity, options]
extensions:
  graphql-analyzer:
    lint:
      rules:
        requireIdField: [warn, { fields: ["id", "uuid"] }]

# Object syntax
extensions:
  graphql-analyzer:
    lint:
      rules:
        requireIdField:
          severity: warn
          options:
            fields: ["id", "uuid"]
```

## Rule name format

Rule names use **camelCase** in configuration:

| Config name             | Rule                    |
| ----------------------- | ----------------------- |
| `noAnonymousOperations` | no-anonymous-operations |
| `noDeprecated`          | no-deprecated           |
| `redundantFields`       | redundant-fields        |
| `requireIdField`        | require-id-field        |
| `uniqueNames`           | unique-names            |
| `unusedFields`          | unused-fields           |
| `unusedFragments`       | unused-fragments        |
| `unusedVariables`       | unused-variables        |
| `operationNameSuffix`   | operation-name-suffix   |
